# Korea Weather

대한민국 행정구역을 기준으로 현재 위치와 검색한 장소의 날씨를 확인하고, 자주 보는 지역을 즐겨찾기로 저장해 상세 날씨까지 확인할 수 있는 날씨 웹 애플리케이션입니다.

## 프로젝트 실행 방법

### 1. 환경 변수 설정

기상청 단기예보 API를 사용하기 위해 공공데이터포털 서비스 키가 필요합니다.

```bash
cp .env.example .env
```

`.env` 파일에 아래 값을 입력합니다.

```env
VITE_KMA_SERVICE_KEY=YOUR_DATA_GO_KR_SERVICE_KEY
```

### 2. 의존성 설치

```bash
npm install
```

### 3. 개발 서버 실행

```bash
npm run dev
```

기본적으로 Vite 개발 서버가 실행되며, 터미널에 출력된 로컬 주소에서 확인할 수 있습니다.

### 4. 프로덕션 빌드

```bash
npm run build
```

### 5. 빌드 결과 미리보기

```bash
npm run preview
```

## 현재 위치 사용 안내

앱은 브라우저의 Geolocation API를 사용해 현재 위치 기반 날씨를 조회합니다.

- 앱 시작 후 실제 현재 위치를 우선 확인하고, 위치 확인이 지연되면 서울특별시 기준 날씨를 먼저 표시합니다.
- 이후 현재 위치 좌표를 가져오면 화면을 즉시 현재 위치 기준으로 갱신하고, 주소 라벨은 뒤에서 보완합니다.
- 위치 권한이 없거나 현재 위치를 확인할 수 없으면 서울특별시 기준 날씨를 유지합니다.
- macOS 환경이나 브라우저 개발자도구의 모바일 에뮬레이션 환경에서는 현재 위치 조회가 실패할 수 있습니다.

## 구현한 기능에 대한 설명

### 1. 현재 위치 기반 날씨 조회

- 브라우저의 Geolocation API를 사용해 현재 위치를 가져옵니다.
- 위치 확인이 늦어질 때는 서울특별시 기준 날씨를 먼저 보여주고, 이후 실제 위치가 확인되면 즉시 갱신합니다.
- 현재 위치 좌표를 먼저 반영하고, 행정구역 라벨은 reverse geocoding으로 뒤에서 보완합니다.
- 현재 위치 좌표를 바탕으로 현재 기온, 체감 온도, 최고/최저 기온과 24시간 기온을 확인할 수 있습니다.

### 2. 대한민국 행정구역 검색

- 로컬 `korea_districts.json` 데이터를 바탕으로 대한민국 행정구역 트리를 구성합니다.
- 검색창에서 시/군/구/동 단위까지 자동완성으로 지역을 찾을 수 있습니다.
- 선택한 행정구역의 대표 좌표를 조회한 뒤 해당 지역의 날씨를 보여줍니다.

### 3. 24시간 기온 예보 제공

- 메인 페이지와 즐겨찾기 상세 페이지 모두 동일한 24시간 기온 UI를 사용합니다.
- 현재 기온, 당일 최저/최고 기온과 함께 시간대별 기온을 카드 형태로 확인할 수 있습니다.
- 모바일에서는 섹션 내부에서만 가로 스크롤되도록 처리해 전체 페이지 레이아웃이 깨지지 않도록 구성했습니다.

### 4. 즐겨찾기 기능

- 검색으로 선택한 장소만 즐겨찾기에 추가할 수 있으며, 현재 위치는 즐겨찾기 대상에서 제외했습니다.
- 즐겨찾기는 최대 6개까지 저장할 수 있습니다.
- 저장한 즐겨찾기는 이름 수정, 삭제 확인창, 상세 페이지 이동을 지원합니다.
- 즐겨찾기 데이터는 `localStorage`에 저장되어 새로고침 후에도 유지됩니다.

### 5. 즐겨찾기 상세 페이지

- 선택한 즐겨찾기 장소의 핵심 날씨 지표를 별도 페이지에서 확인할 수 있습니다.
- 현재 기온, 당일 최저/최고 기온, 강수 확률, 풍속, 시간대, 좌표, 업데이트 시각과 24시간 기온을 제공합니다.
- 메인 페이지와 시각적으로 통일된 레이아웃을 적용해 화면 전환 시 흐름이 자연스럽도록 구성했습니다.

### 6. 반응형 UI

- 데스크톱과 모바일 화면 모두에서 사용할 수 있도록 레이아웃과 카드 크기를 조정했습니다.
- 모바일에서 발생할 수 있는 가로 overflow를 방지하고, 필요한 영역만 내부 스크롤을 허용했습니다.

## 확인 방법

- `npm run build`로 타입 검사와 프로덕션 빌드를 확인할 수 있습니다.
- 브라우저에서 위치 권한을 허용했을 때 현재 위치 기준 날씨로 전환되는지 확인할 수 있습니다.
- 위치 권한을 거부하거나 위치 확인이 실패했을 때 서울특별시 기준 날씨로 유지되는지 확인할 수 있습니다.
- 검색으로 지역을 선택한 뒤 즐겨찾기 추가, 이름 수정, 삭제 확인창, 상세 페이지 이동 흐름을 확인할 수 있습니다.

현재 자동 테스트 스크립트는 별도로 두지 않았고, 빌드 검증과 브라우저 수동 시나리오 확인을 기준으로 동작을 검증했습니다.

## 기술적 의사결정 및 이유

### 1. TanStack Query를 사용한 서버 상태 관리

- 날씨 정보와 행정구역 데이터는 서버 상태에 가깝기 때문에 `TanStack Query`로 관리했습니다.
- 좌표와 시간대를 query key에 포함해 위치별 데이터를 안정적으로 캐싱할 수 있도록 했습니다.
- 날씨 데이터는 짧은 시간 내에 과도하게 다시 요청하지 않도록 `staleTime`을 설정했습니다.

### 2. 현재 위치는 지연 폴백 + 즉시 좌표 반영 전략으로 처리

- 현재 위치를 무조건 오래 기다리면 첫 화면이 늦어지기 때문에, 짧은 시간 동안 실제 위치를 우선 시도하고 늦어지면 서울특별시 기준 날씨를 먼저 보여주도록 구성했습니다.
- 좌표를 확보하면 주소 라벨 조회보다 먼저 날씨를 갱신해 체감 속도를 높였습니다.
- 권한 거부, 시간 초과, 위치 확인 실패 같은 브라우저 환경 차이를 메시지와 함께 처리해 첫 진입 경험이 끊기지 않도록 했습니다.

### 3. 기상청 API 응답을 앱 전용 모델로 변환

- 기상청 초단기실황, 단기예보 API는 응답 형식이 복잡하고 좌표도 격자 좌표 변환이 필요합니다.
- API 응답을 그대로 UI에서 처리하지 않고, `current`, `daily`, `hourly` 형태의 앱 전용 모델로 변환해 화면 로직을 단순하게 유지했습니다.
- 요청 간 최소 간격과 재시도 로직을 두어 공공 API 사용 시 발생할 수 있는 제한 상황에 대응했습니다.

### 4. 행정구역 검색은 로컬 JSON 기반으로 처리

- 검색 시마다 외부 API를 호출하는 방식보다, 정적인 대한민국 행정구역 데이터를 앱에 포함하는 방식이 검색 응답 속도와 안정성 면에서 유리하다고 판단했습니다.
- 초기 로딩 후에는 빠른 자동완성과 트리 탐색이 가능해 사용자 경험이 더 자연스럽습니다.

### 5. 즐겨찾기는 Context + localStorage 조합으로 구현

- 즐겨찾기는 로그인 없이 브라우저 단위에서 유지되면 충분한 기능이기 때문에 별도 백엔드 없이 구현했습니다.
- React Context로 전역 접근성을 확보하고, `localStorage`와 함께 사용해 페이지 이동 및 새로고침 이후에도 동일한 상태를 유지하도록 했습니다.
- 저장 개수 제한, 중복 저장 방지, 잘못된 데이터 정제 로직, 삭제 확인 흐름을 함께 두어 상태 품질을 보완했습니다.

### 6. 메인/상세 공통 UI 컴포넌트 재사용

- 시간대별 기온 영역과 정보 카드 UI를 공통 컴포넌트로 분리해 메인 페이지와 상세 페이지에서 같은 패턴을 재사용했습니다.
- 이를 통해 디자인 일관성을 높이고, 이후 UI 수정이 필요한 경우 한 곳에서 관리할 수 있도록 했습니다.
